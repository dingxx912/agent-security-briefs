# 智能体身份安全 · 技术概要

## 技术栈

- **身份引擎**：身份联邦、去中心化身份(DID)、代理卡(Agent Card)可验证凭证
- **策略引擎**：意图驱动访问控制(IBAC)、自然语言→策略规则转换、策略模拟器（先演练再上线）
- **安全大模型**：意图分析、内容防护、认知与模型引擎
- **审计与溯源**：全链路可观测、TEE 可验证执行、区块链存证
- **接入适配**：兼容 Dify/LangChain、支持 SDK/协议注入/边缘节点/otel 标准集成

## 四层架构

```
┌──────────────────────────────────────┐
│   智能体运营平面（顶层）                │
│   资产管理 │ 费用配额 │ 绩效优化        │
│   安全态势 │ 事件响应 │ 合规报告        │
├──────────────────────────────────────┤
│   安全防护层（第二层）                  │
│   数字身份管理 │ 认知风控管理          │
│   资产权限治理 │ 企业策略管理          │
│   履约合规审计 │ 数据服务防护          │
├──────────────────────────────────────┤
│   核心能力层（第三层）· 四大引擎        │
│   认知与模型 │ 数据与记忆               │
│   身份与策略 │ 连接适配                 │
├──────────────────────────────────────┤
│   立体管控层（底层）                   │
│   统一网关 │ 安全浏览器 │ 沙箱 │ SDK   │
│   流量分析 │ 溯源                       │
└──────────────────────────────────────┘
```

**四层关系**：底层是硬边界（网关+Browser+沙箱），核心能力层是引擎（策略/身份/模型），安全防护层是业务化封装（资产管理/风控/审计），运营平面是统一管控中枢。

## 六维信任框架（核心设计）

| 维度 | 回答的问题 | 实现手段 | 结论 |
|---|---|---|---|
| 身份可信 | 你是谁？ | DID + Agent Card + 统一身份目录 | 管人的方案管不了智能体 → 给智能体发身份证 |
| 意图对齐 | 你想干嘛？ | 策略驱动意图解析 + 三层对齐（接入/行为/编排） | 在越权前拦截，防提示注入与任务劫持 |
| 生成有界 | 你能看/说什么？ | 字段级动态脱敏 + 内容过滤 + 本地知识库强制引用 | 在"看得到"和"说得出"之间建边界 |
| 行为可控 | 你能做什么？ | MCP 安全网关 + 实时授权 + 确定性工作流引擎 | 防止失控的自主行动和无限循环 |
| 链路可审 | 你做了什么？ | 全链轨迹记录 + TEE 可验证证明 | 跨域复杂链路仍能"重放事实" |
| 责权可溯 | 谁负责？ | 区块链存证 + 不可抵赖责任链 | 为合规审计和 AI 保险提供司法级证据 |

## 五大技术壁垒

| 壁垒 | 说明 | 对标情况 |
|---|---|---|
| 混合身份 + 意图合约 | 在"自然人-应用-智能体-模型-工具"全链路上建身份体系 | 国内首发 |
| 动态细粒度鉴权 + 自然语言策略 | 用大白话描述权限，自动生成策略，支持操作/行/字段级控制 | 国内首发 |
| IBAC 意图驱动访问控制 | 将行为映射回任务意图，从"内容安全"跃迁到"任务与行为安全" | 国际前沿 |
| 全域管控平面 | 打通模型路由+工具访问+数据资产，不是单点 LLM 网关 | 业内最全 |
| HITL 人机协作共治 | 阻断/引导/协作/进化四级，人工决策沉淀为策略优化闭环 | 业内最全 |

## 接入方式

| 方式 | 侵入性 | 适用场景 |
|---|---|---|
| 协议注入（反向代理） | 零代码改造 | 快速接入，不想动现有系统 |
| 边缘节点部署 | 零代码改造 | 混合云 / 国产化环境 |
| otel 标准集成 | 零代码改造 | 已有可观测性体系的团队 |
| SDK 嵌入（推荐） | 侵入性小 | 需要更细粒度的控制能力 |

## 当前技术债务/限制

- 对特定小众框架的适配尚需逐个验证
- 自然语言→策略转换的准确性在复杂场景下需人工复核
- TEE 依赖特定硬件，部署成本较高
- 策略模拟器的覆盖场景还在持续扩充中
