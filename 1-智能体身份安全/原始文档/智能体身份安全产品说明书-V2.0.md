## 1. 执行摘要：从「用模型」到「运营智能体」的安全底座

大模型(LLM)的红利正在加速释放，但真正决定企业竞争力的，不再是「有没有模型」，而是**能否安全、可控地大规模运营智能体(Agent)**——让它们持续参与业务决策与执行，而不会制造新的系统性风险。

**智能体安全平台**，是面向中大型企业的**智能体原生(AI-native)安全与治理平台**，以「身份 + 意图 + 行为 + 审计」为主线，提供覆盖从人到智能体、从模型到数据、从工具到资产的全栈安全能力：

- 通过**混合身份与代理卡(Agent Card)**，把自然人、业务系统、智能体、大模型、工具和企业资产串联在同一身份与信任域中；
- 通过**意图驱动访问控制(IBAC)与自然语言策略引擎**，让企业可以用业务语言定义安全边界，并对智能体行为进行持续对齐和动态授权；
- 通过**六维信任框架 + 全链路审计 + 可验证执行 + 上链溯源**，实现从「谁发起、为何发起、做了什么，到结果是否合规」的端到端可观测与责任闭环。

平台既可以作为独立产品落地，也可以与公司现有的终端安全、云安全、网关安全、AI XDR 等产品组合，形成**完整的智能体安全解决方案与安全运营服务**，支撑运营商、金融、能源、政企等行业的 AI 战略升级。

## 2. 为什么现在必须重新设计「智能体安全」？

### 2.1 传统安全范式的失效点

站在国际头部 AI 厂商和安全大厂的视角，智能体带来的核心变化有三点：

1. **执行主体从「人」变成「混合体」**  
   - 过去：用户 → 应用 → 系统，主体清晰、路径有限；  
   - 现在：用户 → 智能体 → （其他智能体）→ 模型 → 工具 → 数据/系统，主体多层嵌套，行为由模型动态生成。

2. **控制边界从「接口」变成「意图与上下文」**  
   - 过去：通过 API 权限、网络边界、账号角色控制；  
   - 现在：同一接口在不同意图与上下文下，风险完全不同。仅凭「接口/字段」做策略，已经不够。

3. **风险从「单点事件」演变为「链路级与系统性风险」**  
   - 提示注入、越权调用、隐性数据泄露、幻觉决策、供应链污染……这些风险往往是在多轮对话、多次工具调用、跨系统链路中逐步累积并放大的。

因此，智能体安全不是在旧有安全堆栈上「打几个补丁」，而是需要一个**围绕智能体重构的安全与治理层**。

### 2.2 六类关键风险与监管趋势

结合国际实践与本地监管方向，可以归纳出六类智能体关键风险，对应我们后续的「六维信任框架」：

1. **身份可信**：谁在代表谁行动？（人 / 智能体 / 第三方模型 / 工具）  
2. **意图对齐**：当前任务究竟在做什么？目标是否被篡改？  
3. **生成有界**：智能体可以看什么、说什么、引用什么？  
4. **行为可控**：它在调用哪些工具、改哪些系统状态、能推演到哪一步？  
5. **链路可审**：从指令到结果，跨域跨系统的链路是否可重放与可证明？  
6. **责权可溯**：出了问题，责任在谁？如何给监管与保险一个可接受的答案？

监管侧（数据安全、生成式 AI 管理办法、关键信息基础设施等）正在对上述问题形成明确要求：**不只是「不能出事」，而是要「说得清、给得出证据」**。

## 3. 产品定位：企业级「智能体安全与可信运营」中枢

### 3.1 核心定位

> **为企业提供一个覆盖全部智能体及其上下游资产的安全与治理中枢，让任何智能体行为都在「有身份、有边界、有记录、有责任」的前提下进行。**

平台不是一个单点检测工具，也不是一个模型服务，而是一个**智能体运行治理平面(Agent Governance Plane)**，对内连接各类业务系统、模型与工具，对外承接用户与上层应用的智能体需求。

### 3.2 目标客户与使用角色

- **目标客户类型**  
  - 运营商、金融、能源、制造、政企等中大型组织；  
  - 正在从「局部 AI 赋能」走向「AI 原生业务」的企业。

- **关键使用角色**  
  - CIO/CTO：希望在不牺牲创新速度的前提下，给全公司智能体应用找一个安全底座；  
  - CISO/安全负责人：需要对所有智能体行为有「看得见、控得住、问得到责」的能力；  
  - 业务负责人：希望快速尝试智能体场景，又不愿被安全条线无限拉扯；  
  - 合规与审计：需要可验证的链路记录与责任链条。

## 4. 整体架构：四层能力堆栈 + 六维信任框架

### 4.1 四层功能架构总览

平台采用自上而下四层架构设计：

1. **智能体运营平面（顶层）**  
   - 为业务、安全与运营团队提供一个统一的「智能体控制中枢」，包含：  
     - 智能体资产管理  
     - 配额和成本管理  
     - 绩效与质量优化  
     - 安全态势感知  
     - 事件响应与协同处置  
     - 合规与审计报告中心  

2. **安全防护层（第二层）**  
   围绕「锚定、联动、全链」三个关键词划分为三大板块：  
   - **数字身份管理（锚定）+ 资产和权限治理**  
   - **认知风控管理（联动）+ 企业策略管理**  
   - **履约和合规审计（全链）+ 数据服务和防护**

3. **核心能力层（第三层）**  
   由四大引擎构成：  
   - 认知与模型引擎：安全大模型、意图分析、内容防护  
   - 数据与记忆引擎：记忆知识防护、行为分析引擎、知识库扫描  
   - 身份与策略引擎：身份联邦与解析、动态策略、人机协作  
   - 连接适配引擎：协议转换、协议注入、多框架适配  

4. **立体管控层（底层）**  
   - 智能体统一网关集成管控平面：Agent 协作、模型路由、工具访问、资产与数据接入、CUA/UI 交互；  
   - 企业级安全浏览器、执行沙箱与 SDK、智能体流量分析与溯源，作为「最后一道硬边界」。

### 4.2 六维信任框架（黄金法则）

在上述架构之上，我们将自身对智能体原生威胁的洞察与 Gartner TRiSM、可验证执行、MCP 协议等实践融合，形成六维信任框架：

1. **身份可信：你是谁？（人类用户与智能体）**  
   - 能力：身份联邦与代理卡，为人和智能体建立统一身份目录；  
   - 实现：为每个智能体颁发可验证的「代理卡(Agent Card)」，基于去中心化身份(DID)实现全局可信互认与能力发现；  
   - 结论：解决「管人的方案管不了智能体」的问题。

2. **意图对齐：你想干嘛？谁派你来的？**  
   - 能力：策略驱动的意图解析，在执行前对自然语言任务进行语义分析与策略校验；  
   - 实现：结合接入一致性、行为一致性、编排一致性三层对齐；  
   - 结论：在越权前拦截，防止提示注入与任务劫持。

3. **生成有界：你能看/说什么？**  
   - 能力：细粒度数据护栏 + 内容合规防护；  
   - 实现：字段级动态脱敏、内容过滤策略、本地知识库强制引用；  
   - 结论：让智能体在「看得到」和「说得出」之间建立清晰边界，减少数据泄露与违规输出。

4. **行为可控：你能做什么？计划是？**  
   - 能力：实时策略执行与流程编排；  
   - 实现：通过 MCP 安全网关等硬逻辑，对每次工具/API 调用进行实时授权；结合确定性工作流引擎，约束行为路径；  
   - 结论：防止失控的自主行动和无限循环。

5. **链路可审：你做了什么？**  
   - 能力：全链路可观测与可验证证明；  
   - 实现：记录从用户指令、智能体思考链、工具调用到最终输出的完整轨迹，结合可信执行环境(TEE)生成不可篡改的执行证明；  
   - 结论：在复杂跨域链路下，仍然能够「重放事实」。

6. **责权可溯：这件事谁负责？**  
   - 能力：基于区块链的审计溯源与责任链条；  
   - 实现：将关键操作日志、策略执行结果和可验证证明上链存证，形成不可抵赖的责任链；  
   - 结论：为智能体专项保险与合规审计提供「司法级证据」基础。

## 5. 核心能力与技术壁垒（站在头部厂商视角的差异点）

### 5.1 混合身份与意图合约：身份体系重构（国内首发）

- **痛点反思：**  
  传统企业 IAM/零信任方案是为「人和应用」设计的，对多层嵌套的智能体调用链先天缺位。
- **我们的设计：**  
  - 在「自然人-业务应用-智能体-大模型-工具-企业资产」全链路上，建设混合身份体系；  
  - 创新提出「意图合约」，将一次任务意图绑定在完整调用链上。  
- **领先性：**  
  对标头部 AI 厂商的「system message / policy / sandbox」思路，我们提供的是一个对企业友好的、本地可控的**身份+意图信任层**。

### 5.2 动态细粒度鉴权 + 自然语言策略（国内首发）

- **痛点反思：**  
  静态策略和权限表无法覆盖智能体实时生成的复杂行为，「先上线再补策略」经常成为事故起点。
- **我们的设计：**  
  - 支持用自然语言描述权限需求，自动生成标准化策略规则；  
  - 支持操作级、行级、字段级等多层级动态策略配置；  
  - 提供策略模拟器与影子模式，先演练再切换生效。  
- **领先性：**  
  让业务和安全团队可以在不学习策略语言的情况下管理复杂智能体策略，使「策略即代码(Policy-as-Code)」真正落地。

### 5.3 意图驱动访问控制(IBAC)（业内首发）

- **痛点反思：**  
  仅通过敏感词或 URL 过滤无法判断「这个行为在当前任务意图下是否合理」。
- **我们的设计：**  
  - 将智能体每一步行为都映射回上层任务意图；  
  - 对不符合意图、风险过高或模糊的行为触发人机协同机制；  
  - 将「无意图不安全」作为第一原则。  
- **领先性：**  
  从国际最佳实践看，这是从「输入内容安全」跃迁到「任务与行为安全」，属于智能体时代真正的纵深防御。

### 5.4 全域管控平面与自动资产发现（业内防护最全）

- **痛点反思：**  
  全球范围内，影子智能体与影子 AI 应用已经成为新一代「影子 IT」问题。
- **我们的设计：**  
  - 构建智能体统一管控平面，打通模型路由、工具访问与数据资产；  
  - 提供自动 AI 资产发现与注册能力，形成动态资产清单；  
  - 联动 AI XDR、安全浏览器、终端安全，实现端到端防护。  
- **领先性：**  
  与仅提供单点「LLM 网关」的方案相比，我们提供的是覆盖整个 AI 资产与调用链的**全域防护体系**。

### 5.5 HITL 人机协作共治（AI 原生，业内最全）

- **痛点反思：**  
  国际头部厂商普遍采用红队/安全评估 + 策略更新，但对企业内部细粒度场景的持续人机协同能力不足。
- **我们的设计：**  
  - 提供阻断、引导、协作、进化四级人机协作机制；  
  - 将每一笔人工决策沉淀为强化学习与策略优化的数据闭环；  
  - 支持业务、安全、合规多角色协同参与。  
- **领先性：**  
  将 HITL 从「审核」升级为「共治」，使企业可以在保持创新速度的同时，保持关键风险在「人」手中。

### 5.6 生态兼容与低侵入集成（生态兼容性最好）

- **痛点反思：**  
  国际方案往往假定客户完全基于某一云厂商或某一框架，而国内大型企业普遍是「多云 + 自研 + 多框架」的混合形态。
- **我们的设计：**  
  - 兼容 Dify、LangChain 等主流智能体框架；  
  - 支持 oTel、SDK、协议注入、边缘节点等多种接入方式；  
  - 面向混合云与国产化环境，提供标准化适配与服务。  
- **领先性：**  
  在多云与复杂遗留系统环境下，更贴近国内大型客户现实。

## 6. 典型应用场景与价值闭环

### 6.1 运营商：智能金库审批与高敏数据访问

- 多级审批 + 意图合约 + 全链审计，实现敏感数据访问「事前可控、事中可见、事后可查」。

### 6.2 大型集团：智能体资产发现与准入治理

- 自动发现影子智能体，统一纳管与策略下沉，降低系统性风险。

### 6.3 高监管行业：合规审计与责任链条

- 全链记录 + TEE 可验证执行 + 上链溯源，为监管报送、司法举证和 AI 保险提供基础。

### 6.4 端侧与 GUI 智能体：从单机安全到人机界面安全

- 通过企业级安全浏览器、CUA/GUI 网关，将终端安全能力延伸到「交互界面级智能体」。

### 7.1 商业模式

- 平台授权：按智能体规模、节点数量或功能模块授权；

### 7.2 产品演进（对标头部厂商的自我要求）

- 短期：完善混合身份、IBAC、HITL 等核心能力，打磨 2–3 个标杆行业场景；  
- 中期：增强 eBPF 等深度洞察手段，形成跨层协同检测与阻断能力；  
- 长期：与更多开源/商业智能体框架、标准协议深度对接，参与行业智能体安全标准制定。

## 8. 总结：让智能体成为「可用且可控」的生产力

智能体会像微服务、移动应用一样，成为企业 IT 的基础形态之一。差别在于：**它更强大，也更不可预测**。

智能体安全平台的目标不是「压制智能体」，而是为它们提供一个**被约束、可审计、可进化**的运行环境——让企业可以放心地把更重要的工作交给智能体，同时在任何时候都能回答监管、股东和用户的关键问题：

> 这件事是谁干的？  
> 他/它为什么这么做？  
> 这样做合不合理？  
> 如果不合理，我们能不能追溯并修正？

在这个意义上，智能体安全平台不只是一个安全产品，而是企业 AI 化道路上的「第二安全操作系统」。
