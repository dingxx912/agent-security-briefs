# 智能体身份安全

## 一句话定位

面向中大型企业的智能体原生（AI-native）安全与治理平台，以「身份 + 意图 + 行为 + 审计」为主线，覆盖从人到智能体、从模型到数据、从工具到资产的全栈安全能力。

## 解决什么问题

传统 IAM/零信任方案是为"人和应用"设计的，对多层嵌套的智能体调用链先天缺位。

三大核心变化：
1. 执行主体从"人"变成"人+智能体+模型+工具"的混合体
2. 控制边界从"接口权限"变成"意图与上下文"
3. 风险从"单点事件"变成"链路级系统性风险"

不是给旧安全栈"打补丁"，而是需要一个**围绕智能体重构的安全与治理层**。

## 目标用户

- **客户类型**：运营商、金融、能源、制造、政企等中大型组织
- **关键角色**：
  - CIO/CTO：要安全底座不拖慢创新
  - CISO：要看得见控得住
  - 业务负责人：要快速试 AI 场景不被安全卡死
  - 合规审计：要可验证的责任链

## 核心能力

- **混合身份体系**：为每个智能体颁发可验证的"代理卡(Agent Card)"，自然人、业务系统、智能体、大模型、工具统一在同一身份域
- **意图驱动访问控制(IBAC)**：不是看"你是谁"，而是看"你想干什么"——每一步行为都映射回上层任务意图，偏离即拦截
- **六维信任框架**：身份可信 → 意图对齐 → 生成有界 → 行为可控 → 链路可审 → 责权可溯
- **自然语言策略引擎**：业务团队用大白话描述权限需求，自动生成策略规则，支持策略模拟器先演练再上线
- **HITL 人机协作共治**：阻断/引导/协作/进化四级机制，关键决策在"人"手中，每次人工决策沉淀为策略优化闭环
- **全域管控平面**：统一管控模型路由、工具访问、数据资产，联动 AI XDR、安全浏览器、终端安全

## 典型场景

| 行业 | 场景 | 价值 |
|---|---|---|
| 运营商 | 智能金库审批与高敏数据访问 | 意图合约 + 全链审计，敏感操作事前可控、事后可查 |
| 大型集团 | 影子智能体发现与准入治理 | 自动纳管，策略统一下沉，降低系统性风险 |
| 高监管行业 | 合规审计与责任举证 | 全链记录 + TEE + 上链存证，满足监管报送和司法举证 |
| 端侧/GUI 智能体 | 从单机安全到交互界面安全 | 企业级安全浏览器 + CUA 网关，管到 GUI 操作级 |

## 商业模式

- 平台授权：按智能体规模、节点数量或功能模块
- 行业解决方案：面向运营商、金融、能源等场景化方案包
- 安全运营服务：红蓝对抗、策略运营、元身份托管等持续服务
