## PR 发布流程

1. 热点出现 → 市场部查本表匹配产品
2. 从对应产品的 `market-brief.md` 取金句和角度
3. 如需技术细节 → 从 `architecture-brief.md` 取
4. 找产品负责人确认技术表述无误
5. 发布后把案例补录到对应产品的 `market-brief.md`「过往 PR 案例」栏

## 禁忌

- 不要预测未发生的安全事件
- 不要直接点名竞品
- 不要在产品未发布时承诺具体交付时间
- 技术细节以 `architecture-brief.md` 为准，不要发挥
