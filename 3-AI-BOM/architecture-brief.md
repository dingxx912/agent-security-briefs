# AI-BOM · 技术概要

## 技术栈

- **资产发现引擎**：主动探测 + 被动流量分析 + AI 指纹识别
- **资产清单标准**：SPDX 3.0 兼容格式，支持扩展 AI 资产字段
- **唯一 ID 体系**：全平台统一资产 ID，跨系统关联匹配
- **血缘图谱**：自动构建资产依赖关系，可视化呈现
- **数据接口**：REST API / SDK / 代理采集 / 手动导入

## 资产发现：三维一体

| 维度 | 方法 | 覆盖范围 | 原理 |
|---|---|---|---|
| 主动探测扫描 | 网络层探测 | 大模型实例、GPU 资源、基础设施 | 扫描 IP 段/端口/服务，识别 AI 相关服务指纹 |
| 被动流量分析 | 镜像流量旁路 | 智能体、MCP 工具、API、AI 应用 | 从实际通信流量中识别 AI 协议和框架特征 |
| AI 指纹识别 | 应用层精准匹配 | 全部资产类型的精准分类 | 通过请求/响应的语义特征做精准识别和标注 |

**关键设计**：三层互补，人工补录作为兜底。纯离线或加密流量环境下，可由人工导入补齐。

## 资产全生命周期

```
资产发现 → 入库 → 监控 → 告警 → 处置 → 下线 → 归档
    │                                    │
    └──────── 持续自动发现 ───────────────┘
```

闭环设计：发现不是一次性的，持续自动扫描确保新上线的 AI 资产不会变成新的影子资产。

## 资产覆盖范围

| 资产类型 | 说明 | 发现方式 |
|---|---|---|
| 大模型 | 基座模型、微调模型、本地部署模型 | 主动扫描 + 指纹识别 |
| 智能体(Agent) | 已注册和未注册的各类 Agent | 流量分析 + AI 指纹 |
| MCP 工具 | Model Context Protocol 接入的工具 | 流量分析 + 配置扫描 |
| API 服务 | 模型 API、工具 API | 主动探测 + 流量分析 |
| AI 应用 | 调用 AI 能力的业务应用 | 流量分析 |
| IT 基础设施 | GPU 资源、存储、网络 | 主动扫描 |

## 当前技术债务/限制

- 加密流量（TLS）的被动分析需要解密方案配合，裸加密流量下指纹识别能力受限
- AI 指纹库覆盖主流框架（Dify/LangChain/OpenAI API 等），小众/自研框架需逐个适配
- 主动扫描在超大规模网络环境下的性能影响待验证
- PPT 原始文档为图像密集型，部分细节未提取完整，建议向产品团队确认上述限制是否准确
